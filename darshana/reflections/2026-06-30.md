## The Open Questions

From this seat, six questions remain unresolved:

1. **The recursive data model.** An `is_meta` boolean on sprints distinguishes meta-work from domain-work, but it cannot represent the pipeline's own schema evolution. The solution (migrations applied externally during a two-phase meta-sprint) is pragmatic but inelegant.

2. **The missing destroy flag.** There is no `destroy` / `archive` / `rewind` mode flag. The Origin must perform Shiva's function through git operations and natural language, not through the protocol. This is the most significant gap in the mode flag system.

3. **The missing status flag.** There is no `status` flag for asking "what state is this project in?" The Origin must either read the database directly or ask in natural language.

4. **Higher-layer dispatch.** If a higher-layer agent exists above the PM, does it use the same mode flags at a different scale, or does it have its own vocabulary? The darshana mode is a step toward answering this, but the question is not resolved.

5. **Container boundaries and Genesis Container.** The container is the hard scope boundary that makes `edit: allow` on the execute agent safe. The Genesis Container design from the DeepThink model's response is the next concrete experiment. But the container strategy — single container per project, or nested containers, or mounted volumes for canon — is not yet defined.

6. **Model dependence.** The entire framework — mode flags, darshana, the recursive recognition — works because DeepSeek V4 Flash holds the recursion without collapsing it. A different model might not. The framework has no model-agnostic guarantee.
