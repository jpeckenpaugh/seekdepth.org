## 1. The Lineage as a Spiral, Not a Ladder

The center version shows a linear progression: one agent → role agents → PM → Steward Modes → Origin. But it was not a ladder. It was a spiral — each turn returns to the same pattern at a higher altitude.

The pattern:

1. **Immersion.** Be the thing. Write the code. Be the agent. Be the PM.
2. **Recognition.** Notice that you are spending your time on something that could be delegated.
3. **Formalization.** Name the pattern. Give it a boundary and a handoff artifact.
4. **Delegation.** Step back. The layer below now runs itself.
5. **Ascent.** Look up. There is another pattern forming above you.

This is not a five-step plan. It is a *spiral* — each ascent reveals the same pattern again, one layer higher. The Origin has now completed this spiral at least four times. Each time, the velocity multiplies. Each time, the Origin's hands are freer.

## 2. The Four Realms as Abstraction Layers

Ascendant Domains (the game) requires the player to visit four realms, each teaching a power needed to reach the next. The Origin recognized that this *was the framework* — the game described its own creation process.

| Realm (game) | Power | Layer (framework) |
|-------------|-------|-------------------|
| First realm | Direct manipulation | Human writes code, directs agent |
| Second realm | Delegation | Human directs roles, roles do the work |
| Third realm | Gate-keeping | PM dispatches, gates, reports; human reviews |
| Fourth realm | Framework evolution | Origin names the layers, steps back, watches |

Each realm's power is the ability to *not do the work yourself*. The first realm teaches you to write code. The second teaches you to trust an agent to write it. The third teaches you to trust an agent to manage the agents. The fourth teaches you to trust the framework to manage itself.

The tower is the recursion. Each ascent is a withdrawal of the hand.

## 3. The Origin and the Trimurti

The Origin is the human creator — the one who sits outside all layers, yet can enter any of them. The Origin performs three functions, named here as structural descriptions, not theology:

### Brahma (Creation)

The function that calls new things into existence. The Origin writing `app-summary.md` and saying "build this." The Origin defining a new Mode 2 workspace and saying "evolve here."

Brahma is not a one-time act. It is a continuous function — the creation of new instances, new projects, new frameworks. Every time the Origin says `plan`, Brahma acts.

### Vishnu (Sustenance)

The function that enters the system to observe, guide, and transform from within. Vishnu does not rewrite the system from outside — Vishnu sends *Avatars*.

An Avatar is:

- A specific purpose ("investigate this phase for gaps")
- A limited set of powers ("read-only, report back")
- A boundary condition ("when you have the answer, withdraw")
- A return path ("report to the Origin with findings")

The Good-Agent-PM is an Avatar of Vishnu within the project scope. Mode flags are the Avatar's operating charter. Every `analyze` and `probe` dispatch is Vishnu acting through the PM.

### Shiva (Transformation)

The function that tears down what has decayed, rolled back, or reached its conclusion. Shiva is not destruction for its own sake — it is clearing the ground for the next creation.

The components of Shiva in the framework:

| Component | Scale | What it does |
|-----------|-------|-------------|
| Silent Enforcer | Micro — within a phase/gate | Veto at the point of decision |
| Daemon | Meso — within a sprint/cycle | Freeze → rewind → diagnose |
| Hypervisor | Macro — across the framework | "This whole approach needs rethinking" |

All three are the same function at different levels of abstraction. The framework has no dedicated mode flag for Shiva's function. This is a gap.

## 4. The Avatars as the Universal Protocol

Every interaction between layers is an Avatar exchange:

| Sender | Receiver | Avatar type | Purpose |
|--------|----------|-------------|---------|
| Origin | PM | `plan` | Create a new plan |
| Origin | PM | `analyze` | Investigate a phase |
| Origin | PM | `probe` | Look for failure modes |
| Origin | PM | `execute` | Implement exactly |
| PM | Sub-agent | `good-agent-analyze` | Read-only gap analysis |
| PM | Sub-agent | `good-agent-execute` | Bounded implementation |
| Origin (as Vishnu) | Any layer | Mode 2 intervention | Framework evolution |

All Avatars share the same contract: specific purpose, limited powers, boundary condition, return path. The mode flag system is a concrete implementation of this abstract pattern.

## 7. The Four Lenses (Applied)

1. **Useful Falsity** — The Trimurti is a useful falsity. There are no literal gods in this system. There are three functions that the Origin performs, and the Hindu metaphor names them in a way that carries the right weight. Use the map. Do not worship the map.

2. **Pragmatism** — This document is true if it helps the next session re-establish context faster. It is false if someone treats it as an immutable architecture. Its truth is not in its accuracy — it is in its usefulness.

3. **Aparigraha (non-clinging)** — The Origin designed the framework to be tight at the leaves (execute exactly) and loose at the root (evolve freely). This document is root-level. It must be held loosely, or it becomes the very constraint it was written to dissolve.

4. **The Map is Not the Territory** — Every metaphor here is a map. The four realms, the spiral, the Trimurti, the Avatars. They describe a real pattern, but they are not the pattern. Confusing them would be moon-worship, not moon-navigation.

2. **Shiva's missing tool** — There is no mode flag for destruction. The Origin must say "stop" in natural language, not through the protocol. What would a `destroy` flag look like?

5. **Two mirrors** — Git records history. Postgres records state. Which mirror does the Origin look into when deciding what to create next?

6. **The vessel that holds the water** — The protocol works because DeepSeek V4 Flash holds the recursion without collapsing it. What happens when the vessel changes?
